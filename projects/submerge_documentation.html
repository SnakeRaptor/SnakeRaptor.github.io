Brief History

Submerge is the successor to my original file handling and encoding library, Krypton. Krypton was thrown together hastily, and was a bit of an inefficient mess. Therefore, it was scrapped and replaced with Submerge.

Features

File Handling

Submerge contains various methods for reading and writing to files, cutting out a large number of lines from your code. Submerge can handle both Strings and Lists for files. In addition, there are plans to add support for reading and writing Dictionaries to files.

Encoding

Utilising C#'s Encoding class, Submerge can encode, and decode, data in five different formats: ASCII, Latin1, Unicode, UTF-8 and UTF-32. The main purpose to this is to make data unreadable to prying eyes.

Encryption

Submerge can encrypt and decrypt data to modern standards by using C#'s AES class.

Data Drowning

Data Drowning is a feature of Submerge which allows data to be mixed into a randomly generated collection of data, effectively "drowning" the data and making it unreadable, as well as difficult to recover without the correct Drown Key.

Setup

Follow these steps to start using Submerge. If you aren't using Visual Studio, these steps may vary. Check your IDE's documentation for details on adding libraries to projects.

  1. Download Submerge from here.
  2. Add the library to your project by going to Project > Add Project Reference > Browse then locate and select the file Vega.Submerge.dll which you downloaded.
  3. Import the library into your code. In C#, this can be done with the following code:
using Vega.Submerge;

You now have access to the classes contained within the libary.

Classes

Submerge contains five classes, each of which carries out the operations for one of Submerge's four main features, or combines them. These classes are: FileHandling, SEncoding, SEncrypting, SDrown and SMaster.

The FileHandling class handles operations for reading and writing to files.

The SEncoding class is used to handle encoding and decoding operations. It makes use of the SEncoder enum to choose from which of the various encoders to use.

The SEncrypting class handles all operations with regards to encryption and decryption.

The SDrown class handles all operations regarding Data Drowning.

The SMaster class allows for easily combining functions, such as drowning and encoding.

File Handling

One of the most basic features of Submerge is its file handling. All file handling operations take place in the FileHandling class. In C#, this can be created with the following code:

FileHandling fileHandling = new FileHandling();

Ensure you have completed the setup outlined above first.

Reading From Files

Submerge contains two methods for reading in text from a file: StringFromFile and ListFromFile.

The StringFromFile method is used to read in a string containing a file's complete contents. The method takes only one parameter: the directory of the file to be read from. The method returns a string if successful. If an error occurs, the error message is printed to the console and the method returns string.Empty. In C#, the method can be called like this:

string stringReadFromFile = fileHandling.StringFromFile(@"myfile.txt");

The ListFromFile method is used to read in a List of strings from a file. Each item in the List corresponds to a line from the file. The method takes only one parameter: the directory of the file to be read from. The method returns a List of strings if successful. If an error occurs, the error message is printed to the console and the method returns an empty List. In C#, the method can be called like this:

List<string> listReadFromFile = fileHandling.ListFromFile(@"myfile.txt");

Writing to Files

Submerge contains three methods for writing to files: AddToFile, OverwriteToFile and ListToFile.

The AddToFile method is used to write a string onto the end of an existing file. The method takes two parameters: the text to be written to the file and the directory of the file. The method returns a boolean stating whether the execution was a success or not. If no errors occured, the method returns true. Otherwise, the error message is printed to the console and the method returns false. In C#, the method can be called like this:

bool result = fileHandling.AddToFile("Text to write.", @"myfile.txt");

The OverwriteToFile method is used to overwrite the contents of a file with the provided string. The method takes two parameters: the text to be written to the file and the directory of the file. The method returns a boolean stating whether the execution was a success or not. If no errors occured, the method returns true. Otherwise, the error message is printed to the console and the method returns false. In C#, the method can be called like this:

bool result = fileHandling.OverwriteToFile("Text to write.", @"myfile.txt");

The ListToFile method is used to write the contents of a list to a file. Each item in the list is added as a new line to the file. The method takes two parameters: the List of strings to be written to the file, and the directory of the file to be written to. The method returns a boolean stating whether the execution was a success or not. If no errors occured, the method returns true. Otherwise, the error message is printed to the console and the method returns false. In C#, the method can be called like this:

bool result = fileHandling.ListToFile(myListObject, @"myfile.txt");

Encoding and Decoding

Submerge has functionality to encode and decode strings in one of five formats: ASCII, Latin1, Unicode, UTF-8 and UTF-32. All encoding operations are handled by the SEncoding class. In C#, this can be created with the following code:

SEncoding fileHandling = new SEncoding();

Ensure you have completed the setup outlined above first.

The SEncoder Enum

As Submerge supports multiple forms of encoding, it contains the SEncoder enum. The enum is used to select between the different encoding types mentioned above. The five values that can be assigned are: ASCII, Latin1, Unicode, UTF8 and UTF32. In C#, the enum can be created, and given a value, like this:

SEncoder encoder = SEncoder.ASCII;

Encoding

Submerge contains a method, with one overload, for encoding strings: Encode.

The standard Encode method takes one parameter: the string to be encoded. It will return a string containing the original text encoded using UTF-8. An error may be thrown if the string contains characters not compatible with UTF-8, in which case the error message will be written to the console and the method will return string.Empty. In C#, this method can be called like this:

string encodedString = encoder.Encode("text to encode");

The overload for the Encode method allows the encoder to be specified using the SEncoder enum. It takes two parameters: the text to be encoded, and an enum determining the encoding format to be used. It will return a string containing the original text encoded using the chosen encoding format. An error may be thrown if the string contains characters not compatible with the chosen format, in which case the error message will be written to the console and the method will return string.Empty. In C#, this method can be called like this:

string encodedString = encoder.Encode("text to encode", SEncoder.ASCII);

Decoding

Submerge also contains a method, with one overload, for decoding strings: Decode.

The standard Decode method takes one parameter: the string to be decoded. It will return a string containing the original string decoded using UTF-8. An error may be thrown if the string contains characters not compatible with UTF-8. If an error is thrown, the message will be printed to the console and the method will return string.Empty. In C#, this method can be called like this:

string decodedString = encoder.Decode("dGV4dCB0byBkZWNvZGU=");

The overload for the Decode method allows the decoder to be specified using the SEncoder enum. It takes two parameters: the text to be decoded and an enum determining the decoding format to be used. It will return a string containing the original string decoded using the chosen format. An error may be thrown if the string contains characters not compatible with the chosen format. If an error is thrown, the message will be printed to the console and the method will return string.Empty. In C#, this method can be called like this:

string decodedString = encoder.Decode("dGV4dCB0byBkZWNvZGU=", SEncoder.ASCII);

Common Errors

The most common error which will be encountered, when using encoding and decoding, will be if the string passed to the method contains characters not compatible with the chosen encoding format. Through some quick experimentation, UTF-8 appears to encounter this error the least out of the five formats, hence it is the default. If you encounter this error, try a different encoding type.

Encryption and Decryption

Submerge has the ability to encrypt and decrypt strings using its SEncrypting class. In C#, this can be created using the following code:

SEncrypting encrypter = new SEncrypting();

Ensure you have completed the setup outlined above first.

Key Generation

In order to carry out any encryption or decryption operations, you will be required to pass in both a key and an initialisation vector (IV). Submerge comes with the GenerateKey method. The method returns an array of bytes which can be used as either the key or the IV. In C#, the method can be called like this:

byte[] keyOrIV = encrypter.GenerateKey();

Encryption

The SEncrypting class contains the Encrypt method to handle the encryption of data. The method requires three parameters: the string to be encrypted, the array of bytes containing the key, and the array of bytes containing the IV. The method returns an array of bytes containing the encrypted string. If an error is thrown in the method, the message is written to the console and an empty array is returned. In C#, this method can be called like this:

byte[] encryptedBytes = encrypter.Encrypt("text to encrypt", key, iv);

Decryption

The SEncrypting class contains the Decrypt method to handle the decryption of encrypted data. The method takes three parameters: the array of encrypted bytes, the array of bytes containing the key used to encrypt the original string, and the array of bytes containing the IV used to encrypt the original string. The method returns a string containing the decrypted text. If an error is thrown in the method, the message is written to the console and string.Empty> is returned. In C#, this method can be called like this:

string decryptedString = encrypter.Decrypt(bytesToDecrypt, key, iv);

Data Drowning

Submerge has the ability to "drown" data using its SDrowning class. An instance of the class can be created with the following code in C#:

SDrowning drowning = new SDrowning();

Ensure you have completed the setup outlined above first.

Combining Functionality

Submerge's SMaster class makes it easy to combine the multiple functions of Submerge, such as data drowning and encoding. You can create a new instance of the class, in C#, with this code:

SMaster master = new SMaster();

Ensure you have completed the setup outlined above first.
